### Fundamental Security Design Principles ###
1. Economy of mechanism: security desgins in hardware and software should be as small and simple as possible, making it easier to test and verify
2. Fail-safe Defaults: Security Controls show default to a safe result if conditions fail or an edge case is reached
3. Complete Mediation: everything a resource is requested for, it should be followed with an attempt to authentication and authorize this access
4. Open Design: security should not be implemented through obscurity
5. Separation of Privilege: No one resource, entity, or person should have the keys to the kingdom, instead, power should be separated into different resources
5. Least Privilege: only give resources, endpoints, and entity the minimun and necessary permission in order for them to operate
6. Least Common Mechanism: Minize the utilization of shared resources and accounts in an environment
7. Psychological Acceptability: Software should be easy and intuitive to understand for a user
8. Modularity: security if provide in parts that can be scaled up and down independently.
9. Least astonishment: component of a system should behave in a way that most users will expect it to behave
